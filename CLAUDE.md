## Project Overview

Intelligent B2B prospecting platform for the laser technology industry. Aggregates research funding opportunities from NSF (USA), NIH (USA), and CORDIS (EU) via public APIs, then applies NLP/LLM-based intent classification to let users query prospects conversationally in French.

## Running the Application

There is no build step. Each Python file is a standalone Streamlit app:

```bash
# Primary production app (Google Gemini backend)
streamlit run chatbotAPI.py

# Ollama/local-LLM variant (requires Ollama running locally)
streamlit run app_chatbot2.py

# Rule-based fallback — no external LLM required
streamlit run chatbot_nonllm.py

# LLM benchmark suite
streamlit run benchmark_llm.py
```

No `requirements.txt` exists. Inferred dependencies: `streamlit`, `pandas`, `requests`, `ollama`, `google-generativeai`, `python-dotenv`.

## Environment

Create a `.env` file at the repo root:
```
GEMINI_API_KEY=<your_key>
```

Loaded via `python-dotenv` at startup. Ollama-based variants require Ollama running locally on the default port.

## Architecture

### Data Pipeline

`run_pipeline()` in each app fetches and merges prospects from three public APIs:
- **NSF** — `GET https://api.nsf.gov/services/v1/awards.json`
- **NIH** — `POST https://api.reporter.nih.gov/v2/projects/search`
- **CORDIS** — `GET https://cordis.europa.eu/api/search/results`

Results are normalized into a shared schema (`source`, `title`, `organization`, `country`, `budget_usd`, `contact_name`, `contact_email`, `score`, `keywords_matched`, `description`, `end_date`, `link`) and cached to `leads_laser.csv`.

### Relevance Scoring

Each prospect is scored (0–15) by keyword matching across title + description:
- `femtosecond` = 5 pts, `ultrafast` = 4 pts, `ablation` = 4 pts, `photonics` = 3 pts, `laser` = 2 pts, `optics` = 1 pt

### Intent Classification

User natural-language queries are sent to the LLM (Gemini or Ollama) with a few-shot prompt that returns structured JSON:
```json
{
  "intent": "search" | "reset" | "unsupported",
  "keywords": [],
  "country": "USA" | "France" | "Germany" | "UK" | null,
  "organization_contains": null,
  "min_budget": null,
  "max_budget": null,
  "min_score": null,
  "sort_by": "budget" | "score" | null,
  "sort_order": "asc" | "desc" | null,
  "top_k": 10,
  "reply": null
}
```

Filters from successive turns are merged into `st.session_state` (cumulative context). Rule-based shortcuts handle greetings and resets before the LLM is called.

### Codebase Variants

| File | LLM Backend | Status |
|---|---|---|
| `chatbotAPI.py` | Google Gemini 2.0 Flash Lite | Primary/production |
| `app_chatbot2.py` | Ollama (Mistral/Phi) | Main alternative |
| `app_llm_demo.py` | Ollama, lightweight | Demo / dual-interface |
| `app_off.py` | Offline rule-based | Offline variant |
| `chatbot_nonllm.py` | None (pure rules) | Fallback |
| `app_chatbotlast.py` | Ollama | Earlier iteration |
| `benchmark_llm.py` | Multiple (comparison) | Benchmarking only |

All UI text, prompts, and comments are in **French**.

### Data Persistence

- `leads_laser.csv` — cached prospect database generated by `run_pipeline()`
- `benchmark_llm_results.csv` / `benchmark_llm_summary.csv` — LLM perf metrics
- No external database; all session state lives in `st.session_state`

### API Rate Limiting

A 0.3 s sleep is applied between API requests inside the collection loops to respect public API limits. Do not remove this.
